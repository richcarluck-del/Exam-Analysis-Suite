# 预处理器测试 Web UI 任务列表

- [ ] 任务1: 为新的 Web UI 设置基础项目结构。
  - [ ] 子任务1.1: 创建一个新的目录 `preprocessor_test_ui`。
  - [ ] 子任务1.2: 在 `preprocessor_test_ui` 内部，创建一个简单的 FastAPI 后端 (`main.py`)。
  - [ ] 子任务1.3: 在 `preprocessor_test_ui` 内部，使用 Vite 创建一个新的 React 前端应用（例如，在一个 `frontend` 子目录中）。

- [ ] 任务2: 设计并确认 UI 原型。
  - [ ] 子任务2.1: 使用一个快速的 UI 组件库（如 Material-UI 或 Ant Design）构建一个静态的、无功能的 UI 布局原型。
  - [ ] 子任务2.2: 将 UI 原型的截图或可交互链接呈现给用户，并获取用户的认可。

- [ ] 任务3: 实现后端 API 接口。
  - [ ] 子任务3.1: 创建一个 GET `/api/providers` 接口，用于从 `exam_analysis.db` 数据库中获取所有 API 供应商。
  - [ ] 子任务3.2: 创建一个 GET `/api/models` 接口，用于获取所有 LLM 模型。
  - [ ] 子任务3.3: 创建一个 GET `/api/prompts` 接口，用于从数据库中获取所有 `extract_content_v*` 提示词。
  - [ ] 子任务3.4: 创建一个 WebSocket 接口 `/ws/run-test`，用于处理测试执行请求并将日志流式传输回客户端。

- [ ] 任务4: 在确认 UI 原型后，构建完整功能的前端 UI 组件。
  - [ ] 子任务4.1: 创建用于从下拉列表中选择 API 供应商和模型的组件。
  - [ ] 子任务4.2: 创建用于输入目录的组件。
  - [ ] 子任务4.3: 创建用于选择提示词并在只读区域显示其文本的组件。
  - [ ] 子任务4.4: 创建用于选择测试模式（真实/模拟）的组件，并根据选择显示“模拟运行步骤”勾选区。
  - [ ] 子任务4.5: 在“模拟运行步骤”勾选区中，将步骤 3、5、6 渲染为置灰不可选状态，步骤 1、2、4 渲染为可选状态。
  - [ ] 子任务4.6: 创建“开始测试”按钮。
  - [ ] 子任务4.7: 创建能够处理流式文本的日志显示区域。

- [ ] 任务5: 实现前端应用逻辑。
  - [ ] 子任务5.1: 页面加载时，从后端获取供应商、模型和提示词下拉列表的数据。
  - [ ] 子任务5.2: 为所有用户选择实现状态管理（例如，使用 `useState`, `useReducer`）。
  - [ ] 子任务5.3: 实现客户端逻辑，以便在点击“开始测试”按钮时连接到 WebSocket 并发送完整的测试配置负载。
  - [ ] 子任务5.4: 实现客户端逻辑，以从 WebSocket 接收日志消息并将其附加到日志显示区域。

- [ ] 任务6: 在后端实现对 `preprocessor/main.py` 的调用封装。
  - [ ] 子任务6.1: 在 WebSocket 处理器中，从客户端接收测试配置 JSON。
  - [ ] 子任务6.2: **核心任务**：编写一个“参数转换”函数，该函数严格地将前端发送的 JSON 配置，转换为 `preprocessor/main.py` 所支持的命令行参数列表。此函数**严禁**包含任何业务逻辑，仅做参数映射（例如：将 `mocked_steps: [1, 4]` 转换为 `--real-steps 2`）。
  - [ ] 子任务6.3: 使用 `asyncio.create_subprocess_exec` 调用 `preprocessor/main.py`，并传入上一步转换好的参数列表。
  - [ ] 子任务6.4: 异步地从子进程的 `stdout` 和 `stderr`逐行读取，并通过 WebSocket 立即将每一行发送给客户端，实现日志的实时流式传输。
