# 预处理器测试 Web UI 验证清单

- [ ] **核心原则验证**: 后端服务是否仅作为命令行参数的“翻译层”，而没有自行实现任何 `preprocessor` 的核心业务逻辑。
- [ ] UI 原型设计遵循了简单、大方的原则，并获得了用户的认可。
- [ ] 当选择“模拟测试”时，UI 上的标签是否变为“如果选择模拟测试，请勾选需要 **模拟运行** 的步骤:”。
- [ ] 当选择“模拟测试”时，步骤 3、5、6 的勾选框是否被正确地置灰且不可交互。
- [ ] 当选择“模拟测试”时，步骤 1、2、4 的勾选框是否保持可选状态。
- [ ] 后端能否根据勾选的“模拟步骤”，正确地反向计算出 `--real-steps` 参数并执行命令（例如，勾选了步骤1，则命令中应包含`--real-steps 2 4`）。
- [ ] Web UI 能否成功获取并显示 API 供应商列表。
- [ ] 选择一个供应商后，是否能正确地过滤模型下拉列表并显示供应商的 endpoint 和脱敏后的 API 密钥。
- [ ] Web UI 能否成功获取并显示所有的 `extract_content_v*` 提示词列表。
- [ ] 选择一个提示词后，其完整的文本内容是否能在指定区域正确显示。
- [ ] 输入目录字段的默认值是否为 `D:\10739\Exam-Analysis-Suite\preprocessor\my_test_images`。
- [ ] “开始测试”按钮能否正确地在后端触发测试运行。
- [ ] `preprocessor` 进程的实时日志是否能被流式传输到 UI 的日志区域并正确显示。
- [ ] 这个新的 Web UI 测试工具的创建和使用，是否对现有的 `run_pipeline.py` 或 `preprocessor/main.py` 的命令行执行方式没有产生任何功能性影响。
