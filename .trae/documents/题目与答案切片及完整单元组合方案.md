# 题目与答案切片及完整单元组合方案

## 背景

当前系统已经能够识别题目区域和答案区域，但存在以下问题：
1. 没有物理切片题目和答案区域
2. `complete_units.json` 中缺少题目图片路径
3. 没有生成组合后的完整单元图片
4. 混合模式（题目答案同页）的处理逻辑不完善

## 目标

1. **物理切片**: 在框图的同时，物理切片题目和答案区域
2. **完整单元组合**: 生成组合后的完整单元图片（题目 + 答案）
3. **混合模式支持**: 正确处理混合模式（题目答案同页）
4. **数据关联**: 在 `complete_units.json` 中记录所有切片路径

### 1. 目录结构设计

#### 关键问题：在哪个图片上物理切片？

**当前数据来源**：
- **题目纸（1_corrected.jpg）**：包含题目 1-15
- **答题纸（a_corrected.jpg）**：包含涂卡区 + 第 22、23 题作答区域

**切片规则**：
- 题目切片：从对应的 `source_corrected_image`（矫正后的图片）上切片
- 答案切片：从对应的 `source_corrected_image`（矫正后的图片）上切片

#### 关键问题 2：切片扩展策略（容错设计）

**问题**：大模型识别的坐标可能不够精确，边框可能贴得太紧，导致切掉部分内容。

**解决方案**：向外扩展少量像素，提高容错性。

**扩展像素值**（经验值）：
- **上边**：扩展 10-20 像素（避免包含上一题）
- **下边**：扩展 20-30 像素（防止切掉公式下标、化学式下标等）
- **左边**：扩展 10-15 像素
- **右边**：扩展 10-15 像素

**代码实现**：
```python
def crop_with_padding(image, points, padding=None):
    """
    带扩展的切片函数
    
    Args:
        image: 原图
        points: 坐标点 {top_left, top_right, bottom_right, bottom_left}（归一化坐标，0-1000）
        padding: 扩展像素 [top, bottom, left, right]，默认 [15, 25, 15, 15]
    
    Returns:
        切片后的图片
    """
    if padding is None:
        padding = [15, 25, 15, 15]  # 上、下、左、右
    
    h, w = image.shape[:2]
    
    # 将归一化坐标转换为像素坐标
    x_min = int(min(points['top_left'][0], points['bottom_left'][0]) / 1000 * w)
    y_min = int(min(points['top_left'][1], points['top_right'][1]) / 1000 * h)
    x_max = int(max(points['top_right'][0], points['bottom_right'][0]) / 1000 * w)
    y_max = int(max(points['bottom_left'][1], points['bottom_right'][1]) / 1000 * h)
    
    # 应用扩展（确保不超出图片边界）
    x_min = max(0, x_min - padding[2])  # 左
    y_min = max(0, y_min - padding[0])  # 上
    x_max = min(w, x_max + padding[3])  # 右
    y_max = min(h, y_max + padding[1])  # 下
    
    # 切片
    crop = image[y_min:y_max, x_min:x_max]
    
    return crop
```

**优势**：
- ✅ 容错性提高：即使坐标有偏差，也能保证完整包含题目内容
- ✅ 避免切掉重要信息：特别是公式、化学式的下标等
- ✅ 对后续识别影响小：即使包含相邻题目的部分文字，大模型仍能正确识别本题内容
- ✅ 行业最佳实践：OCR 和文档处理领域的通用做法

**注意事项**：
- 扩展像素值不宜过大（建议 10-30 像素），避免包含过多干扰内容
- 向下扩展多于向上：公式、化学式的下标通常在下方
- 确保不超出图片边界（使用 max/min 函数限制）

```python
def extract_and_crop_questions(content_output_path, workspace_dir):
    for part in content_results:
        sheet_id = part['sheet_id']
        # 从矫正后的图片上切片

            # 跳过涂卡区（涂卡区单独处理）

            cv2.imwrite(output_path, crop, [cv2.IMWRITE_JPEG_QUALITY, 95])
            
            # 记录切片路径和来源图片
            question['question_slice_path'] = output_path
            question['source_image_for_crop'] = corrected_image_path  # 记录从哪张图片切的
```

```python
def generate_complete_unit_images(linked_results, workspace_dir):
    """
    生成完整单元组合图片
    """
    for q_num, data in linked_results.items():
        question = data.get('question', {})
        answer = data.get('answer')
        answer_source = data.get('answer_source')
        
        question_slice_path = question.get('question_slice_path')
        answer_slice_path = data.get('answer_slice_path')  # 非选择题的答案
        
        # 判断模式
        if question.get('is_mixed_sheet'):
            # 混合模式：题目答案同页
            # 直接使用题目切片（因为答案就在题目旁边）
            unit_image_path = question_slice_path
        elif answer_source == 'answer_card':
            # 选择题：题目 + 涂卡答案
            # 组合题目图片和答案标注
            unit_image_path = combine_question_and_answer_card(
                question_slice_path, 
                answer,
                workspace_dir,
                q_num
            )
        else:
            # 非选择题：题目 + 答案（拼接）
            unit_image_path = combine_question_and_answer(
                question_slice_path,
                answer_slice_path,
                workspace_dir,
                q_num
            )
        
        # 记录完整单元图片路径
        data['complete_unit_image_path'] = unit_image_path
```

**组合方式**:
1. **选择题**: 题目图片 + 答案文字标注（右上角或底部）
2. **非选择题**: 题目图片 + 答案图片（上下拼接或左右拼接）
3. **混合模式**: 直接使用题目切片（答案已在旁边）

### 4. 混合模式处理逻辑

#### 4.1 混合模式识别

在**步骤 2（页面分类）**中，识别混合模式：
```python
if sheet_type == 'mixed':
    is_mixed_mode = True
    # 标记该页既包含题目也包含答案
```

#### 4.2 混合模式的数据流

```
步骤 4: 内容提取
  ↓ 识别题目区域（包含答案）
  ↓ 物理切片到 question_slices/
  
步骤 5: 合并结果
  ↓ 按题号分组
  ↓ 标记 is_mixed_mode = true
  
步骤 6: 完整单元生成
  ↓ 判断 is_mixed_mode
  ↓ 如果是混合模式：直接使用题目切片作为完整单元图片
  ↓ 否则：组合题目和答案
```

#### 4.3 混合模式的特殊情况

**跨页题目**:
- 如果题目跨越多页（如第 22 题分两栏）
- 需要拼接多个切片
- 生成一个完整单元图片

**处理方式**:
```python
def combine_cross_page_questions(question_fragments, workspace_dir, q_num):
    """
    拼接跨页的题目切片
    """
    crops = []
    for fragment in question_fragments:
        crop = cv2.imread(fragment['question_slice_path'])
        crops.append(crop)
    
    # 左右拼接（假设是分栏）
    combined = cv2.hconcat(crops)
    
    # 保存
    output_path = f"{workspace_dir}/complete_unit_images/{sheet_id}/CU{q_num}.jpg"
    cv2.imwrite(output_path, combined)
    
    return output_path
```

## 注意事项

1. **切片命名唯一性**: 使用 `sheet_id + 题号` 确保命名唯一
2. **目录不存在时创建**: 切片前检查并创建目录
3. **图片格式统一**: 统一使用 JPG 格式，质量 95%
4. **切片扩展策略**: 使用 `crop_with_padding` 函数，默认扩展 [15, 25, 15, 15] 像素
   - 向上扩展 15 像素
   - 向下扩展 25 像素（防止切掉公式下标）
   - 向左扩展 15 像素
   - 向右扩展 15 像素
   - 确保不超出图片边界
5. **跨页处理**: 需要特殊处理跨页题目（拼接多个切片）
6. **混合模式识别**: 依赖步骤 2 的页面分类结果
7. **答案提取**: 非选择题的答案提取可能需要额外的 OCR 或大模型识别
